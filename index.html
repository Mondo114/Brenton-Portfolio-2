About Me
I’m a freelance UX Designer and Front End Developer pursuing a full-time position. I currently live in Redondo Beach, located in the greater Los Angeles area. I earned my Bachelor of Science degree in Food Science from Cal Poly, San Luis Obispo.
Transitioning into tech, I first earned my Full Stack Web Development certificate from UCLA Extension. With this foundation in coding, I have continued my education, gaining skills in UI/UX. I recently earned the UI/UX Design Specialization Certificate from Cal Arts, in addition to UX-related certifications from other programs. I am currently working on an all-new UX Design portfolio website to showcase my UX-focused work.
I studied Food Science so I could combine science and creativity in product development. As the Research and Development Manager for a large snack foods company, I took products from ideation to launch. My management background cultivated my comprehensive approach to problem solving and provided me with an aptitude for understanding the needs of clients and customers.

Board Game Discovery
  • What: Board Game Discovery is a web app that helps you figure out which board game to play that makes the most sense in your given situation.
  • Why: If you have a group of people coming over for a game night and you want to know what new game to play, this app allows you to search by different attributes to help narrow it down. You may search by 1 attribute at a time. The reason there are a few dozen games and not thousands in the database is because each game has been entered by hand, allowing for the exact attributes we wanted to include (we couldn’t find already existing APIs that had all the attributes we wanted).
  • Who: This app was built by 3 people; I built the Front End
  • Time: The majority was built in about 1 week, but has been improved over time
  • Technologies: Node.js, jQuery, MySQL, and several npm modules

Frij'it
  • What: Frij’it is a web app to start your day. It’s a collection of customizable widgets (“magnets”) that appear on your fridge (“frij”). You can choose which news, sports, and special interest articles to display. Also available are widgets for stocks, weather, and the time. Ideally there would be dozens of widgets with plenty of customization options.
    *If the news articles don’t populate from the dropdown, it’s because the API has hit its limit of uses for the day (APIs are not currently paid for).*
  • Why: I’ve always found it a hassle to check all your news sources, stocks, etc. in separate places. This app organizes them all into one place.
  • Who: This app was a 4-person effort; I created the UI, set up all news-related widgets, and worked on various other parts
  • Time: The majority was built in about 1 week, but has been improved over time
  • Technologies: Multiple APIs, Node.js, React, MongoDB, and several npm modules

Firefly Memory Game
  • What: Firefly Memory Game is a web game. The object is to click on each character once. Each time you click on a character, all twelve characters randomly scramble their position. When you have successfully clicked on each character exactly once, you win.
  • Why: I wanted a relatively simple app to start out with for learning React. I thought it would be fun to try out an interactive game to demonstrate React’s functionality.
  • Who: Web game was built by me
  • Time: The majority was built in about 1 week, but has been improved over time (mostly aesthetics)
  • Technologies: Node.js, React, and several npm modules
Firefly Memory Game GitHub Repository
Vampire Companion
  • What: Vampire Companion is a fun web app, which seeks to find which vampire character you most identify with. This app provides you with a survey, and based on your responses, yields your “vampire companion”.
  • Why: I wanted an interactive way to demonstrate working with JSON and APIs. I felt using vampires in a matching app was more fun and humorous than using regular people.
  • Who: Web app was built by me
  • Time: The majority was built in about 1 week, but has been improved over time (mostly aesthetics)
  • Technologies: Node.js, my own vampire API, JSON, Express, and several other npm modules
